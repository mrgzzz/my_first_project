1. required - обязательное поле, type="email|tel|url" - проверка формата, pattern - регулярное выражение, minlength/maxlength - длина, min/max - числовые диапазоны.
2. novalidate отключает встроенную браузерную валидацию полностью, а через JavaScript можно selectively отключать/включать отдельные проверки.
3. Связывать через for и id или оборачиванием. Важно для скринридеров - озвучивает label при фокусе на поле.
4. email - email, tel - телефон, password - пароль, number - числа, date - даты, url - ссылки.
5. pattern проверяет значение по регулярному выражению. Использовать для кастомных форматов данных.
6. input - при вводе, change - при изменении значения, blur - при потере фокуса. Позволяют валидировать "на лету".
7. Использовать aria-live="polite", aria-describedby для связи с сообщениями об ошибках.
8. fieldset группирует логически связанные поля, legend дает описание группы. Улучшает навигацию и семантику. 
9. Через кастомные JavaScript-функции, сравнивающие значения полей и устанавливающие кастомные специальные сообщения об ошибках.
10. Псевдоклассы :valid, :invalid, :required, :optional и атрибутные селекторы [aria-invalid="true"].
