1. O(n) - линейная сложность, O(n²) - квадратичная. Пример: поиск в массиве - O(n), пузырьковая сортировка - O(n²)
2. Бинарный поиск делит отсортированный массив пополам на каждой итерации, поэтому сложность O(log n)
3. Преимущества: простота реализации. Недостатки: медленная работа на больших массивах
4. Проверять делители только до квадратного корня числа
5. Рекурсивная может привести к переполнению стека, итеративная эффективнее по памяти
6. Console.log, точки останова в DevTools
7. Алгоритм Евклида использует остаток от деления, эффективен из-за быстрого уменьшения чисел
8. Пустые массивы, массивы с одним элементом, отрицательные числа
9. Использовать Set или объект для хранения уникальных значений
10. Читаемые имена переменных, избегание дублирования кода, комментарии
