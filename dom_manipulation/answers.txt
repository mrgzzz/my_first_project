1. innerHTML парсит HTML и может создавать уязвимости, а textContent работает только с текстом и безопаснее.
2. Делегирование событий - это когда обработчик вешается на родительский элемент и обрабатывает события дочерних элементов через всплытие.
3. getElementById ищет по ID, querySelector находит первый элемент по селектору, querySelectorAll находит все элементы по селектору.
4. Стандартное поведение предотвращается методом event.preventDefault() когда нужно отменить действие по умолчанию.
5. classList.add() добавляет класс не затрагивая другие, а className полностью перезаписывает все классы элемента.
6. Новый элемент создается через document.createElement() и добавляется методами appendChild() или insertBefore().
7. Всплытие событий - это когда событие поднимается от целевого элемента к корню документа, а stopPropagation() останавливает этот процесс.
8. Данные из формы получаются через FormData объект или прямой доступ к value элементов формы.
9. input срабатывает при любом изменении, change при потере фокуса после изменения, keyup при отпускании клавиши.
10. Производительность оптимизируется минимизацией операций с DOM, использованием фрагментов и кэшированием ссылок на элементы.
