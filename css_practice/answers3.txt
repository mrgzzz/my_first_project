Ответы на вопросы
1. ID специфичнее класса (100 vs 10), но классы можно переиспользовать, а ID только один на странице.

2. Каскад применяет стили по порядку: браузерные → пользовательские → авторские, при конфликте побеждает более специфичный селектор.

3. Специфичность: ul li.important = 0,1,2; #site-title = 1,0,0; .content > .featured-post = 0,2,0

4. Наследуются: color, font-family, line-height. Не наследуются: margin, padding, background.

5. > выбирает только прямых потомков, а через пробел - всех потомков на любой глубине.

6. :hover для интерактивности (кнопки, ссылки), :nth-child() для стилизации элементов по порядку (полосатые таблицы).

7. !important принудительно применяет стиль, ломает каскад - используйте только для переопределения сторонних стилей.

8. ::before/::after добавляют декоративный контент: иконки, украшения, текст. Содержат только свой content.

9. При одинаковой специфичности побеждает правило, которое объявлено последнее в CSS-файле.

10. Инспектор элементов показывает примененные стили, специфичность, наследование, позволяет редактировать CSS в реальном времени.
