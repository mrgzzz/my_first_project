1. Node.js - это среда выполнения JavaScript на стороне сервера, которая использует событийно-ориентированную архитектуру и неблокирующий ввод-вывод, что делает ее эффективной для создания масштабируемых сетевых приложений.

2. Простой HTTP-сервер создается с помощью модуля http и метода createServer, который принимает функцию обработки запросов.

3. GET - получение данных, POST - создание новых данных, PUT - обновление существующих данных, DELETE - удаление данных.

4. Middleware - это функции, которые обрабатывают запросы перед их окончательной обработкой, используются для логирования, аутентификации, парсинга тел запросов и т.д.

5. Различные HTTP-методы для одного URL обрабатываются путем проверки свойства method объекта запроса и выполнения соответствующих действий.

6. CORS (Cross-Origin Resource Sharing) - механизм для контроля доступа к ресурсам с разных доменов, соответствующие заголовки нужны для разрешения или ограничения межсайтовых запросов.

7. Тело POST-запроса читается через обработчики событий 'data' и 'end' объекта запроса, где данные собираются по частям и затем парсятся.
