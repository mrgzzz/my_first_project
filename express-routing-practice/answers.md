1. Express.js предоставляет более высокий уровень абстракции по сравнению с чистым Node.js, упрощая создание маршрутов, обработку middleware и организацию кода.

2. Middleware в Express.js - это функции, которые имеют доступ к объектам запроса и ответа и могут выполнять код, вносить изменения в запрос/ответ или завершать цикл запроса-ответа, выполняются в порядке их определения.

3. Модульную структуру роутеров организуют через создание отдельных файлов с роутерами с помощью express.Router() и их подключение в главном файле приложения через app.use().

4. Express.js поддерживает методы роутинга get, post, put, delete, patch и другие, которые соответствуют HTTP-методам и используются для обработки различных типов запросов.

5. Параметры URL обрабатываются через req.params, а query-параметры через req.query в обработчиках маршрутов.

6. app.use() используется для подключения middleware ко всем методам и маршрутам, а app.METHOD() - для обработки конкретных HTTP-методов на определенных маршрутах.

7. Централизованная обработка ошибок организуется через middleware с четырьмя параметрами (err, req, res, next), которое размещается после всех роутеров.

8. router.param() используется для выполнения кода при наличии определенного параметра в URL, позволяя валидировать или предварительно обрабатывать параметры.
