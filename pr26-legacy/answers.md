1. Основные проблемы legacy PHP-кода: SQL-инъекции из-за отсутствия экранирования пользовательского ввода; смешение PHP-логики и HTML-разметки; отсутствие обработки ошибок; небезопасный вывод данных.

2. Преимущества Laravel: структурированная архитектура MVC; встроенные механизмы безопасности; система шаблонов Blade; миграции БД; роутинг; ORM Eloquent; автоматическая инъекция зависимостей.

3. MVC в Laravel: Model (модели) работают с данными БД через Eloquent; View (представления) - Blade-шаблоны; Controller (контроллеры) обрабатывают запросы и связывают модели с представлениями.

4. Composer - менеджер зависимостей для PHP, управляет установкой и обновлением пакетов, автозагрузкой классов.

5. Возможности Blade: наследование шаблонов через @extends; секции @section/@yield; директивы условных операторов и циклов; экранирование вывода; подключение подшаблонов.

6. Маршрутизация в Laravel: определяется в файлах routes/, поддерживает GET, POST, PUT, DELETE методы, параметры маршрутов, группы маршрутов, middleware.

7. Безопасность в Laravel: защита от SQL-инъекций через Eloquent/Query Builder; CSRF-токены; валидация данных; экранирование вывода в Blade; хеширование паролей.

8. Передача данных в шаблоны: через метод view() с массивом данных: view('template', ['key' => 'value']) или compact(); в шаблоне доступ через {{ $key }}.
