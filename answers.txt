Ответы на вопросы
1. Три основные области Git:  
- Рабочая папка — файлы, с которыми вы работаете.  
- Индекс (staging) — область подготовки файлов к коммиту.  
- Репозиторий (history) — база данных с историей коммитов.  

2. Команды:  
- git add — добавляет изменения в индекс.  
- git commit — фиксирует изменения из индекса в репозиторий.  
- git push — отправляет коммиты в удалённый репозиторий.  

3. Информативные коммиты помогают понять историю изменений без просмотра кода.  

4. git add имя_файла — добавляет конкретный файл. git add . — все изменения. Используйте выборочный add для контроля, а git add . для массовых изменений.  

5. git status показывает состояние рабочей папки и индекса (изменённые, добавленные, неотслеживаемые файлы).  

6. git remote add origin URL связывает локальный репозиторий с удалённым. origin — стандартное имя для удалённого репозитория.  

7. Флаг -u устанавливает связь ветки с удалённой, чтобы в дальнейшем использовать git push без указания параметров.  

8. Ветка по умолчанию — main (ранее master). Важно для согласованности в команде.  

9. git log показывает историю коммитов: хеш, автора, дату и сообщение.  

10. Если сделать git commit без git add, изменения не попадут в коммит (только ранее добавленные).
